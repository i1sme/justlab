# justlab — виртуальная химическая лаборатория

## Что это

Веб-приложение для изучения химии: 3D таблица Менделеева, просмотрщик молекул, виртуальный стол для проведения реакций с реалистичной визуализацией. Образовательный продукт.

**Целевая аудитория (3 группы):**

- 🏫 **Школьники** (5-11 класс) — упрощённый UX, школьная программа, гейминг.
- 🎓 **Студенты вузов** — углублённый контент: органика, механизмы, спектры, кинетика.
- 🌱 **Новички / самоучки** — те, кто просто интересуется химией. Нужен onboarding, объяснения "от нуля".

UX поддерживает **два уровня сложности** (школа/вуз) + **режим новичка** с подсказками. Уровень выбирается при первом входе и переключается в настройках.

## Главные принципы (в порядке приоритета)

1. **Доступность важнее красоты.** Если фича работает на iPhone SE 2020 или Android-планшете 2018 года — её можно делать. Если нет — упрощаем, режем, или делаем как опцию для "high"-пресета.
2. **Производительность — это фича.** Performance budget: ≤200 KB gzipped initial JS, ≤3s TTI на 4x throttled CPU, ≥30 FPS на средней 3D сцене на интегрированной графике.
3. **Реалистичность ≠ симуляция.** Мы НЕ считаем квантовую химию. Мы показываем заранее подготовленные/курированные результаты реакций с красивой визуализацией. Честно говорим "неизвестно", если реакции нет в базе.
4. **Прогрессивное улучшение.** 2D работает везде. 3D — где есть WebGL2. WebGPU — где доступен. AR/VR — опционально.
5. **Оффлайн-first.** Приложение должно полностью работать без интернета после первой загрузки.

## Стек

- **Vite + SvelteKit** (Svelte 5 с Runes). React не используем — слишком тяжёлый для целевых устройств.
- **Three.js** напрямую для 3D. Не R3F.
- **RDKit-JS** (WASM) для химии: парсинг SMILES, молекулярные свойства, генерация координат.
- **OpenChemLib-JS** для 2D-структурных формул.
- **Tailwind v4** для стилей.
- **Dexie** (IndexedDB) для журнала и кеша.
- **Comlink** для воркеров.
- **lz-string** для сжатия состояния эксперимента в URL (для шеринга).
- **Vitest** + **Playwright** для тестов.
- **Без бэкенда.** Всё работает локально, шеринг через сериализованный URL hash.

## Архитектурные правила

- **Любой парсинг/расчёт — в Web Worker.** Главный поток только рендерит UI и 3D-сцену. Сообщения через Comlink.
- **3D-объекты создаются через instanced meshes** для одинаковых атомов. Никогда не делать `new Mesh()` в цикле для атомов одного типа.
- **LOD обязателен** для всего, что может появиться в количестве > 50 на сцене.
- **Adaptive quality**: при старте детектится GPU (через `navigator.gpu` / `WEBGL_debug_renderer_info`), выставляется preset. Пресеты: `low` (2D-only или 3D с минимумом), `medium`, `high`.
- **Lazy load всего тяжёлого**: RDKit (~10 MB WASM), Three.js, Rapier — только динамические `import()` по необходимости.
- **`prefers-reduced-motion` уважается всегда** — анимации заменяются на мгновенные переходы.
- **i18n с нулевого дня.** Никаких хардкодных строк в UI. Минимум: `ru`, `en`.

## Структура

```
src/
  lib/
    chemistry/    # RDKit-обёртки, парсеры, БД реакций
    render3d/     # Three.js: scene, materials, shaders, LOD
    render2d/     # canvas/SVG для 2D-структур
    physics/      # Rapier (если используется)
    workers/      # web worker entry points
    storage/      # Dexie схемы
    i18n/         # переводы
    ui/           # Svelte-компоненты
  routes/         # SvelteKit роуты
  data/           # курированная БД реакций и элементов
  shaders/        # GLSL/WGSL
```

## Соглашения по коду

- TypeScript strict.
- Названия атомов/элементов — IUPAC символы (`H`, `Na`, `Fe`), не русские.
- SMILES — primary canonical representation для молекул в БД и API.
- Координаты атомов — в ангстремах (Å) внутри chemistry-слоя; конвертируются в условные единицы сцены в render-слое.
- Цвета атомов — стандарт CPK (Corey-Pauling-Koltun) по умолчанию.

## Что НЕ делать (важно)

- ❌ **Не пытаться симулировать реакции с нуля.** Только база + анимации.
- ❌ **Не использовать R3F / React.** Решение принято осознанно ради bundle size.
- ❌ **Не блокировать главный поток** парсингом или вычислениями.
- ❌ **Не добавлять зависимости больше 50 KB gzipped** без обсуждения.
- ❌ **Не делать фичу, если она ломается на iPhone SE 2020.** Либо упрощать, либо делать опциональной.
- ❌ **Не выдумывать продукты несуществующих реакций.** Лучше честно сказать "реакция не найдена в БД", чем показать ложную химию (это образовательный продукт, доверие критично).

## Источники данных

- **PubChem** (https://pubchem.ncbi.nlm.nih.gov) — публичная БД молекул и реакций. Используется офлайн (импорт нужного среза).
- **PDB** (для больших биомолекул, опционально).
- **GHS pictograms** (https://www.unece.org/trans/danger/publi/ghs/ghs_welcome_e.html) — иконки безопасности.
- Курируемая собственная БД ~500 реакций школьной программы (нужно собрать).

## Тестирование

- **Unit (Vitest)** — для chemistry/, parsers, утилит.
- **E2E (Playwright)** — golden path: открыть таблицу → выбрать элемент → собрать реакцию → увидеть результат.
- **Визуальные тесты** — Playwright screenshots для 3D-сцен (baseline + сравнение).
- **Performance smoke**: Lighthouse CI на main, performance budget enforced.

## Локализация

- Язык интерфейса: `ru` / `en` (минимум). Файлы: `src/lib/i18n/{lang}.json`.
- Названия элементов и веществ: IUPAC + локализованные ("Сера" / "Sulfur").
- Числа, даты — через `Intl`.

## UX-режимы

При первом запуске пользователь выбирает уровень. Уровень управляет:

- глубиной контента (термины, объём объяснений)
- доступными реакциями
- сложностью UI (упрощённая палитра инструментов vs полная)

| Режим       | Аудитория            | Особенности                                                                      |
| ----------- | -------------------- | -------------------------------------------------------------------------------- |
| **Новичок** | Самоучки, любопытные | Onboarding, glossary, всплывающие подсказки, "что это значит?" на каждом термине |
| **Школа**   | 5-11 класс           | Школьная программа, гейминг, безопасные сценарии                                 |
| **Вуз**     | Студенты             | Полная функциональность: спектры, механизмы, кинетика                            |

Уровень переключается в настройках. Состояние сохраняется в IndexedDB.

## Этапы

- **MVP**: таблица Менделеева 3D + SMILES-просмотрщик + 10-20 реакций с анимацией. Режим новичка с onboarding.
- **v0.2**: виртуальный стол, журнал в IndexedDB, 100+ реакций, шеринг через URL.
- **v0.3**: учебные квесты, два уровня (школа/вуз), параметры среды, эффекты.
- **v0.4**: AR-режим (WebXR), оффлайн (service worker), экспорт в PDF.
- **v1.0**: i18n (ru/en), accessibility-аудит, продакшн.
